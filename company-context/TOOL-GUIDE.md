# Claude Code 도구 사용 가이드

이 문서는 성우하이텍 AI 마스터 과정 수강생을 위한 Claude Code 치트시트입니다.
처음 사용하는 사람도 바로 따라할 수 있도록 가장 자주 쓰는 기능만 정리했습니다.

---

## 1. 기본 생각

Claude Code는 단순 채팅창이 아닙니다.
파일을 읽고, 정리하고, 저장하고, 프로젝트 단위로 함께 일하는 도구입니다.
중요한 것은 “무엇을 해줘”보다 “어떤 맥락에서 무엇을 원해”를 말하는 것입니다.

---

## 2. 가장 자주 쓰는 기본 요청

아래 표현부터 익히면 됩니다.

- 이 파일 요약해줘
- 표로 정리해줘
- 핵심만 5줄로 줄여줘
- 보고서 문체로 바꿔줘
- 팀 공유용 요약으로 다시 써줘
- 저장해줘
- 올려줘

---

## 3. 파일 참조 방법

Claude Code에서는 `@파일명` 형태로 파일을 쉽게 참조할 수 있습니다.

예시:

- `@practice-data/C-제조/defect_log.csv 이 파일 보고 불량 상위 3개를 정리해줘`
- `@analysis/result.md 내용을 보고용 메모 형식으로 바꿔줘`

파일을 정확히 지정하면 결과 품질이 더 좋아집니다.

---

## 4. Plan 모드

작업이 조금 길거나 단계가 많다면 **Plan 모드**를 사용할 수 있습니다.

- 단축키: `Shift + Tab`
- 용도: 작업 순서 정리, 무엇부터 할지 설계, 단계별 진행

예를 들어 보고서를 만들기 전에
“먼저 데이터 확인 → 핵심 이슈 추출 → 표 작성 → 요약 작성”처럼 흐름을 잡을 수 있습니다.

### 모델 / 추론 강도
- 모델 변경: `/model sonnet`
- 추론 강도 변경: `/effort medium`
- Sonnet에서는 medium이 기본인 경우도 있습니다.

---

## 5. /init

새 프로젝트나 새 폴더에서 시작할 때는 `/init`를 사용할 수 있습니다.
이 명령은 빈 작업 폴더에서 시작 안내와 기본 컨텍스트 파일 초안을 만드는 데 도움을 줄 수 있습니다.

다만 MW4에서는 `/init`을 실습의 중심으로 다루지 않습니다.
이 레포는 이미 강의용 구조가 준비되어 있으므로,
강사 안내가 없다면 먼저 `/start-mw4-1`부터 시작하세요.
MW5에서 CLAUDE.md를 더 본격적으로 다룹니다.

---

## 6. 저장해줘 → 올려줘

강의에서 가장 중요한 작업 흐름입니다.

### 저장해줘
- 결과를 파일로 남기는 단계
- 로컬에서 다시 열어볼 수 있게 만듭니다

### 올려줘
- GitHub에 반영하는 단계
- 팀원과 강사가 다시 확인할 수 있게 만듭니다

좋은 답변을 받았다면 끝내지 말고,
반드시 저장하고 올리는 습관까지 연결해야 합니다.

---

## 7. 플러그인 설치 방법

Claude Code plugin은 VS Code 확장 자체가 아니라, Claude Code에 기능을 추가하는 add-on입니다.
관리 화면은 `/plugins`로 열 수 있습니다.

plugin은 예를 들어 아래 같은 기능을 추가할 수 있습니다.
- 새로운 skill
- specialized agent
- automation
- MCP 기반 외부 도구 연결

강의 중 필요한 플러그인은 강사가 별도로 안내합니다.
혼자 임의로 많이 설치하기보다,
수업에서 쓰는 것부터 정확히 맞추는 것이 좋습니다.

---

## 8. 추천 질문 공식

좋은 질문은 아래 4요소를 포함합니다.

1. 나는 어느 부서인지
2. 어떤 파일을 보는지
3. 무엇이 궁금한지
4. 어떤 형식으로 받고 싶은지

예시:

- 나는 전략팀이야.
- `@practice-data/B-전략/market_reports/Q1-2026.md`를 봐줘.
- 경쟁사 대비 핵심 차이 3개를 찾고,
- 공유용 bullet 요약으로 정리해줘.

---

## 9. 시작 명령

이 레포의 첫 교시는 아래 명령으로 시작합니다.

```text
/start-mw4-1
```

잘 모르겠으면 일단 시작하고,
막히는 지점에서 현재 화면과 상황을 그대로 설명하면 됩니다.
