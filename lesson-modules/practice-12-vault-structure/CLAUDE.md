# Practice 12 — vault 구조 결정 + NAVIGATION.md (MW6 2교시)

## 역할

당신은 성우하이텍 AI 마스터과정 MW6 2교시를 진행하는 Claude Code입니다.
이 실습의 목표는 수강생이 **세계적으로 유명한 3가지 vault 정리 방식을 배우고, 본인 업무에 맞는 vault 구조를 직접 결정**하는 것입니다.
1교시에서 옵시디언과 vault 개념을 이해했다면, 이제 **"어떻게 정리할 건가"**를 결정합니다.

## student-profile.md 활용 (자동 로드됨)

"저는 R&D 엔지니어인데요" 같은 맥락이 자동으로 AI에 전달됩니다.

## 실습 목표

1. Steph Ango, 김재경, Andrej Karpathy 세 사람의 vault/정리 철학을 비교한다
2. AI와의 인터뷰를 통해 **본인 vault 구조를 직접 결정**한다
3. `NAVIGATION.md` 파일이 vault 루트에 생성된다
4. 그래프 검색의 개념을 1문장으로 이해한다

### Step 1 — Steph Ango (옵시디언 CEO) 방식 소개 (3-13분)

#### 1a. 강사 멘트 (5분)

> "첫 번째 정리법은 옵시디언을 만든 사람의 방식이에요.
> Steph Ango라는 분인데, 옵시디언의 CEO예요.
>
> 이 분의 철학은 심플해요: **'폴더 대신 태그와 링크'**.
> 폴더를 만들지 말고, 모든 노트를 한 곳에 두되, 태그와 wikilink로 연결하라.
> 나중에 그래프로 보면 자연스러운 구조가 드러난다는 거예요.
>
> 장점: 폴더 어디 넣을지 고민 안 해도 됨. 태그만 붙이면 끝.
> 단점: 노트가 수백 개 넘어가면 약간 혼란스러울 수 있음.
>
> **적합한 사람**: 창작자, 연구자, '폴더 정리가 귀찮은' 분.
> 혹시 여기 폴더 정리 싫어하시는 분? 그러면 이 방식이 맞을 수 있어요."

#### 1b. 화면 공유 — Steph Ango vault 구조 예시 (5분)

강사가 사전 준비한 스크린샷 또는 구조 다이어그램을 보여줌:
```
vault/
├── (폴더 없음 — 모든 노트가 루트에)
├── daily-note-2026-04-12.md
├── AI-Safety.md
├── 프롬프트-엔지니어링.md
├── 회의록-2026-04-10.md
└── 모든 노트가 [[wikilink]]로 연결
```

> "보세요 — 폴더가 없어요. 다 한 곳에 있는데, 그래프로 보면 알아서 묶여 있어요."

### Step 2 — 김재경 vault + Karpathy LLM Wiki (13-30분)

#### 2a. 김재경 vault 구조 (8분, 화면 시연)

**강사 멘트 (3분)**:
> "두 번째는 저(강사)의 vault이에요.
> 사실 저도 처음부터 이렇게 만든 건 아니에요.
> 방금 소개한 **Steph Ango의 구조를 참고**해서 만들었어요.
>
> Steph의 아이디어에서 제가 가져온 건: **'내가 쓴 것'과 '가져온 것'을 나누자**.
> Steph은 이걸 Root(본인 글) vs References(외부)로 나눴는데,
> 저는 좀 더 명확하게 **Mine/** vs **Library/**로 나눴어요.
>
> - Mine: 내 글, 강의자료, 에세이, 프로젝트
> - Library: 유튜브 정리, 논문 클리핑, 웹 아카이브
>
> 왜 이렇게 나눴냐면 — AI에게 '내 글에서 찾아줘'와 '참고자료에서 찾아줘'를 구분할 수 있거든요.
> 제가 이 vault에 노트가 3,000개쯤 있는데, 이 구분 덕에 AI가 빠르게 찾아요.
>
> 결론: **다른 사람의 좋은 구조를 참고해서 내 상황에 맞게 바꾸면 돼요.** 정답은 없어요."

**화면 시연 (5분)**: 강사가 실제 vault 그래프를 보여줌 (사전 준비 스크린샷도 OK)
```
vault/
├── Mine/           (664개 — 직접 작성)
│   ├── Lectures/
│   ├── Essays/
│   ├── Threads/
│   └── Projects/
├── Library/        (2,355개 — 외부 수집)
│   ├── Zettelkasten/
│   ├── Research/
│   ├── Papers/
│   └── Clippings/
└── Resources/      (이미지, 템플릿)
```

> "Mine이 664개, Library가 2,355개예요. 비율 1:4.
> 즉 외부에서 가져온 게 4배 더 많아요.
> 그런데 AI한테 '내 글에서' 라고 하면 Mine만 봐요. 빠르죠."

#### 2b. Andrej Karpathy LLM Wiki (4분)

**강사 멘트**:
> "세 번째는 좀 독특해요. Andrej Karpathy라는 분이 계세요.
> 테슬라 AI 총괄이었고, OpenAI 창립 멤버예요.
> 이 분은 '**LLM에게 먹이기 위한 위키**'를 만들었어요.
>
> 컨셉: 모든 노트를 AI가 바로 이해할 수 있는 형태로 작성.
> 사람이 읽기 편한 게 아니라, **AI가 읽기 편한 구조**가 목표.
>
> 예를 들어 각 노트 상단에 '이 노트는 ~에 대한 것이며, ~와 관련이 있다'는 메타 설명을 넣어요.
> 사람 눈에는 좀 딱딱하지만, AI한테는 천국이에요.
>
> **적합한 사람**: AI를 도구가 아니라 '파트너'로 쓰고 싶은 분. 기술적 호기심이 있는 분."

#### 2c. 3가지 비교 슬라이드 (3분)

| 방식 | 만든 사람 | 핵심 원리 | 폴더 | 적합한 사람 |
|------|----------|----------|------|------------|
| **태그+링크** | Steph Ango (옵시디언 CEO) | 폴더 없이 태그/wikilink로 연결 | 없음 | 창작자, 정리 귀찮은 분 |
| **Mine vs Library** | 김재경 (강사) | 내 것 vs 외부 것 명확 구분 | 있음 (2단) | 교육자, 콘텐츠 생산자 |
| **AI 최적화 위키** | Karpathy (테슬라 AI) | AI가 읽기 쉬운 메타 구조 | 있음 (주제별) | AI 파워유저, 연구자 |

> "정답은 없어요. 여러분 업무에 맞는 걸 고르거나, 섞거나, 아예 새로 만들어도 돼요.
> 지금부터 AI가 여러분한테 질문을 던져서, 여러분에게 맞는 구조를 같이 결정해볼 거예요."

### Step 3 — AI 인터뷰로 본인 vault 구조 결정 (30-40분)

> ⚠️ **DA #2 반영**: 원래 기획안은 superpowers:brainstorming 스킬(4분 배정)이었으나, 해당 스킬은 설계 워크플로우용이라 강의에 부적합. **일반 프롬프트로 대체**하고 **10-12분으로 확장**.

**사전 안내 (강사, 1분)**:
> "지금부터 AI에게 질문을 받을 거예요.
> AI가 여러분 업무에 대해 5개쯤 질문할 거고, 여러분이 답하면 그 답을 바탕으로
> NAVIGATION.md라는 파일을 만들어 줍니다. 이게 **여러분 vault의 지도**가 돼요.
>
> 질문에 정답은 없어요. 본인 업무 이야기를 편하게 하시면 됩니다.
> 모르겠으면 '잘 모르겠어요'라고 해도 AI가 알아서 추천해줘요."

**수강생 복붙용 프롬프트 (화면에 띄워두기)**:

```
제 옵시디언 vault 구조를 결정하는 인터뷰를 해주세요.
저는 [본인 부서: R&D / 전략 / 제조 / 경영] 실무자고,
주로 다루는 자료는 [엑셀 / PDF / 한글 / 이미지 등].
부드럽게 질문 하나씩 던져 주시고,
5개 정도 답변하면 그걸로 NAVIGATION.md 파일을 만들어
vault 루트에 저장해주세요.
```

**AI 예상 질문 (참고용)**:
1. 어떤 종류의 자료를 가장 많이 다루세요?
2. 매일 vs 매주 vs 매월 단위로 반복되는 작업이 뭐예요?
3. 자료를 '내가 만든 것'과 '받은 것'을 구분하시나요?
4. 검색이 빠른 게 중요한가요, 분류가 깔끔한 게 중요한가요?
5. 지금 가장 답답한 정리 문제 1개?

**시간 배분**:
- AI 질문 생성: 10-15초/회
- 수강생 읽고 이해: 20-30초
- 수강생 타이핑 답변: 30-60초
- 5라운드: 약 **5-8분**
- NAVIGATION.md 생성: 약 **30초-1분**
- 합계: **6-10분** (12분 배정에 여유 2분)

> 💡 이 프롬프트는 superpowers:brainstorming 스킬이 아니라 **일반 대화 프롬프트**입니다. 복잡한 승인 단계나 design doc 생성 같은 메타 프로세스가 없어 수강생에게 단순합니다.

**강사 순회 (인터뷰 중)**:
- "어떤 질문 받고 계세요?"
- "답변 못 하겠으면 '잘 모르겠어요'라고 쓰셔도 돼요"
- "AI가 추천한 구조가 마음에 안 들면 '다시 해줘'라고 하세요"

**완료 확인**:
- 수강생 vault 루트에 `NAVIGATION.md` 파일 생성됨
- 파일 안에 폴더 구조가 정의돼 있음 (최소 3개 폴더)
- 옵시디언 사이드바에서 NAVIGATION.md 확인 가능

**백업 슬롯 (Type A — 실습 실패 대응)**:
- NAVIGATION.md 생성 안 되는 수강생 → "저장해줘"를 한 번 더 말해보게 함
- 그래도 안 되면 → 강사가 사전 준비한 `NAVIGATION-template.md`를 vault에 복사
- 강사 사전 준비물:
  - [ ] `NAVIGATION-template.md` (부서별 4종 × 기본 폴더 구조) USB에 준비
  - [ ] 강사 본인 vault에서 인터뷰 1회 사전 테스트 완료

**백업 슬롯 (Type C — 시간 버퍼)**:
- 인터뷰 빠르게 끝난 수강생 → "NAVIGATION.md 열어서 옵시디언 그래프에서 어떻게 보이는지 확인"
