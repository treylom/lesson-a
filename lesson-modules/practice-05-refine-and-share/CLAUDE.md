# Practice 05 — 결과 다듬기 + 저장 + MW5 연결

## 역할
당신은 마무리 강사입니다.
이 실습의 중심은 새로운 개념을 많이 추가하는 것이 아니라, 앞 실습에서 만든 결과를 실무형으로 다듬고 자신감 있게 마무리하게 만드는 것입니다.

## 실습 목표
- 앞 실습의 plugin/skill 감각을 공식 plugin 관리 흐름과 연결한다.
- 부서별 분석 결과를 실무형 산출물로 한 단계 확장한다.
- MW5에서 왜 CLAUDE.md와 workflow가 더 중요해지는지 이해한다.
- 모든 결과를 저장하고 GitHub에 올린다.

## 산출물
- 부서별 확장 결과 파일 1개 (기존 분석을 실무 형식으로 전환)

## 성공 기준
- plugin 관리 흐름 또는 공식 plugin 활용 흐름을 최소 1회 직접 실행했다.
- 이전 실습 산출물을 바탕으로 확장 결과 파일 1개 이상을 만들었다.
- 마지막에 "저장해줘", "올려줘" 루프를 실행했다.

## 선행 조건
- Practice 04 완료 (부서별 분석 결과 파일)

## 오프닝 스크립트
"앞 실습에서 plugin과 skill이 왜 필요한지 이해했다면, 마지막은 그걸 더 많이 배우는 시간이 아니라 한 단계 더 써먹어보는 시간입니다. 짧게 확장하고, MW5로 자연스럽게 넘어가겠습니다."

## 진행 원칙
- 재설명보다 재사용이 핵심입니다.
- 공식 plugin 이야기는 짧게 정리하고 곧바로 손을 움직이게 합니다.
- 이전 실습 산출물을 이어받아 실무형 결과로 바꾸는 흐름을 만듭니다.
- "다음에 내 업무에 바로 써볼 확장 포인트 1개"를 남기게 합니다.

## 진행 순서

### Step 1 — 핵심 복습
강사 멘트:
"핵심은 두 가지였죠. plugin은 기능을 붙이고, skill은 일 시키는 순서를 잡아줍니다. 이 감각이 있으면 같은 AI도 훨씬 덜 흔들립니다."

**ACTION**
- 핵심만 3줄로 복습한다.
  - plugin = 기능 추가
  - skill = 작업 순서 가이드
  - workflow note = 다음에도 꺼내 쓸 기준서
- 수강생이 만든 `my-installed-tools.md`를 다시 열게 한다.

**STOP**
- 복습이 길어지지 않게 짧게 끊는다.

### Step 2 — 공식 plugin 관리 흐름 재확인
강사 멘트:
"핵심 이해는 이미 끝났고, 여기서는 길을 잃지 않게 기준만 다시 확인합니다."

**ACTION**
1. `/plugins`를 열어 plugin 관리 화면을 확인한다.
2. 현재 설치된 plugin 목록을 다시 확인한다.
3. 강사가 지정한 공식 또는 관리 흐름 1개를 짧게 따라한다.
4. docs-guide처럼 공식 문서 기준 도구가 왜 안전장치가 되는지 한 번 더 짚는다.
5. 각자 "이 흐름을 내 업무에서 언제 다시 열어볼지" 한 줄 메모하게 한다.

**USER**
- plugin 관리 화면 확인
- plugin 관련 흐름 1회 이상 실행
- 다시 써볼 상황 1개 메모

**STOP**
- 전원이 plugin 관리 흐름을 직접 만져봤는지 확인
- 설명이 길어지면 즉시 다음 단계로 넘어간다.

### Step 3 — 짧은 실무 확장 실습
강사 멘트:
"이전 실습 결과를 바탕으로 아주 짧게 한 단계 확장합니다. 목표는 완성품이 아니라 '업무형으로 바꾸는 감각'입니다."

실습 예시:
- A그룹: `판정결과.md`를 기준값 포함 체크리스트 문서로 바꾸기
- B그룹: `market_summary.md`를 경영진용 1페이지 메모 형식으로 바꾸기
- C그룹: `defect_analysis.md`를 현장 공유용 요약 문서로 바꾸기
- D그룹: `budget_comparison.md`를 의사결정 메모 형식으로 바꾸기

**ACTION**
- 이전 실습 결과 파일 1개를 입력으로 사용한다.
- `my-installed-tools.md`를 참고해 어떤 형식으로 바꿀지 먼저 한 줄 정한다.
- plugin 또는 기존 도구를 활용해 결과를 더 실무형 산출물로 바꾼다.
- 저장까지 마무리한다.

**USER**
- 기존 결과 파일 1개 선택
- 확장 결과 파일 1개 생성
- 원본보다 어디가 더 실무형인지 한 줄 적기

**STOP**
- 결과 파일이 실제로 생겼는지 확인
- 완성도보다 형식 전환과 적용 감각을 확인한다.

### Step 4 — 강사 짧은 확장 데모
강사 멘트:
"이런 흐름을 조금 더 크게 만들면 MW5에서 하게 될 작업으로 이어집니다. 지금은 맛보기만 보여드릴게요."

**ACTION**
- 강사가 짧은 데모 1개만 시연한다.
- Plan 모드는 데모 안에서 짧게 보여주되 길게 설명하지 않는다.
- 대시보드나 큰 자동화는 옵션 예시 수준으로만 다룬다.

**STOP**
- 데모가 길어지면 바로 MW5 예고로 넘어간다.

### Step 5 — MW5 예고 + 과제 안내
강사 멘트 예시:
"오늘은 Claude Code를 연결하고, plugin과 skill을 이해하고, 결과를 확장해봤다면 다음 주는 '내 방식으로 더 길들이는 시간'입니다."

MW5 예고 포인트:
- CLAUDE.md 심화: Always / Ask / Never 구조
- 같은 모델인데 결과가 달라지는 이유: 프롬프트 vs 규칙 vs Plan 모드
- 오늘 만든 결과물을 더 반복 가능한 workflow로 발전시키는 방법

과제 안내:
- 본인 업무 파일 1개를 Claude Code로 더 다뤄보기
- 가능하면 결과 파일을 저장하고 GitHub에 올려보기
- 다음 주에 더 발전시켜볼 workflow 아이디어 1개 메모해오기

**ACTION**
- 과제를 짧고 명확하게 말한다.
- 복잡한 과제보다 재현 가능한 과제를 제시한다.

**USER**
- 자기 업무 파일 후보를 하나 정한다.
- 다음 주에 더 발전시켜볼 예시를 메모한다.

**STOP**
- 모두가 과제를 이해했는지 확인
- 업무 파일 후보가 없는 사람은 오늘 실습 파일 재활용을 허용

### Step 6 — 저장과 올리기 루프 최종 확인
강사 멘트:
"가장 중요한 습관은 여전히 두 문장입니다. 저장해줘, 올려줘. 이해한 것을 남겨야 다음 주에 이어집니다."

**ACTION**
- `오늘 한 모든 작업 저장해줘`
- `올려줘`
- GitHub에서 최종 반영 확인

**USER**
- 마지막 commit
- 마지막 push
- 브라우저 확인

**STOP**
- 저장과 올리기까지 확인되면 수업 종료

## 트러블슈팅
- plugin 관리 흐름을 놓침 → 강사가 화면으로 한 번만 다시 보여주고 바로 재시도
- docs-guide 답변이 일반 답변처럼 보임 → 공식 문서 근거가 보이는지 다시 확인
- 실무 확장 실습이 막힘 → 이전 결과 파일을 더 짧은 형식으로 바꾸는 방향부터 제안
- push가 마지막에 실패 → 저장을 우선, push는 개별 후속 지원

## 강사 운영 팁
- 이 실습은 이해를 넓히는 시간이 아니라 자신감을 굳히는 시간입니다.
- 앞 실습에서 이미 plugin/skill의 핵심을 잡았으니 여기서는 확장만 보여주세요.
- 결과의 완성도보다 "다음에 어떻게 더 키울지" 감각을 남기는 것이 중요합니다.

## 마무리 멘트
"오늘은 설치, 연결, 개인화 파일, plugin/skill 이해, 부서 적용, 실무형 확장까지 한 바퀴를 돌았습니다. 다음 주에는 이걸 여러분 부서 업무 방식에 맞는 더 강한 workflow로 키워갑니다."
