## 실습 개요

MW7은 MW4-6에서 배운 모든 도구를 **본인 업무에 직접 적용**하는 날입니다.

- 10개 과제 중 **본인 부서에 맞는 1개**를 선택합니다
- Level 1 달성이 전원 목표입니다 (수업 가용 75분 기준)
- Level 2, Level 3은 시간이 남는 사람이 이어서 진행합니다
- 팀별 발표는 없습니다. 결과물 URL을 **[Google Sheets 공용 시트](https://docs.google.com/spreadsheets/d/1G4x7lcPy7IWsgr28rtIMAVsI4IO8rUxDoPbtkcTXEBc/edit?usp=drivesdk)** 에 올리는 것으로 마무리합니다
- 강사가 시연 직전 1~2분간 시트를 띄워 부서별 흐름만 가볍게 훑습니다 (개인 발표·개별 평가 없음)

**이 실습에서 Claude가 하는 역할:**
- 어떤 결과물이 나와야 하는지 알려줍니다
- 수강생이 막혀서 물어보면 더 자세히 답변합니다
- 먼저 단계를 풀어서 설명하거나 유도하지 않습니다

## Level 기준

> 각 Level에서 **어떤 결과물**이 손에 쥐어져 있어야 하는지를 기준으로 판단합니다.
> 과정이 아니라 결과물로 Level을 측정합니다.

### Level 1 — 충실히 (전원 목표, 45~60분 소요)

**손에 쥐어야 할 것:**
- 자료조사 결과를 Obsidian에 정리한 노트 폴더 (노트 3개 이상)
- Obsidian 그래프에서 노트들이 연결된 스크린샷 1장
- AI Studio에서 실제 작동하는 웹페이지/대시보드 URL 1개
- Google Sheets 공용 시트 본인 번호 행에 닉네임(익명 가능) + 결과물 URL + 사진 1장 기재 ("Level 1 완료" 표시)

Level 1 완료의 의미: 자료조사 → 정리 → 바이브코딩 한 사이클을 돌았다.

### Level 2 — 고도화 (+30~45분)

**손에 쥐어야 할 것:**
- Level 1 산출물을 본인이 직접 써본 뒤 발견한 "아쉬운 점 3가지" 메모
- Claude Code로 그 아쉬운 점을 개선한 업데이트 버전 URL
- Google Sheets 공용 시트 본인 행에 개선 버전 URL 교체 + "Level 2 완료" 표시

Level 2 완료의 의미: 쓰면서 불편함을 발견했고, 직접 개선했다.

### Level 3 — 확장 (+15~30분)

**손에 쥐어야 할 것:**
- 같은 과제에서 새 기능 1개 이상 추가된 버전 URL
- 또는 공통 주제 풀에서 1개를 더 착수한 결과물
- Google Sheets 공용 시트 본인 행에 확장 버전 URL 교체 + "Level 3 완료" 표시

Level 3 완료의 의미: 본인 업무에 쓸 수 있는 수준으로 확장했다.

## 표준 사이클 — 4단계

> 이 사이클을 한 번 돌면 Level 1 완료입니다.
> 단계별 세부 절차는 수강생이 물어보면 안내합니다.

```
[1단계] 자료조사
   프롬프트 생성기 GPTs → Deep Research → Markdown 파일로 저장

[2단계] 정리
   /knowledge-manager 호출 → Obsidian Library/Research/ 저장 → 그래프 확인

[3단계] 바이브코딩
   10개 중 선택한 과제 → AI Studio 새 프로젝트 → Claude Design 또는 Stitch 택1

[4단계] 산출물 제출
   작동하는 URL → Google Sheets 공용 시트 본인 번호 행에
              닉네임(익명 가능) + URL + 사진 1장 기재
```

각 단계에서 나와야 하는 결과물은 위 Level 기준을 참고하세요.

## /knowledge-manager 안내

> 강사가 먼저 한 번 시연합니다. 시연을 보고 나서 본인 자료로 따라 하세요.

`/knowledge-manager`를 처음 호출하면 중간에 선택지가 4~5개 나옵니다.
처음 보면 낯설어 보이지만, 강사 시연을 본 뒤에는 자연스럽게 선택할 수 있습니다.

**시연 흐름 (강사가 보여주는 순서):**
1. `/knowledge-manager` 입력
2. 중간 선택지 4~5개 — 각 옵션이 무슨 의미인지 설명
3. 자료조사 결과물 1건을 실제로 정리해 Obsidian에 저장
4. Obsidian 그래프 뷰에서 연결 확인
5. "이제 본인 자료로 해보세요" 전환

시연 후 본인이 할 때도 같은 흐름으로 진행하면 됩니다.

**막히면 이렇게 요청하세요:**
```
"내 Deep Research 결과를 Library/Research/MW7-{이름}-과제-{N번}/ 폴더에
항목별로 파일 분리해서 저장해줘."
```

## Google Sheets 공용 시트

**공유 링크**: https://docs.google.com/spreadsheets/d/1G4x7lcPy7IWsgr28rtIMAVsI4IO8rUxDoPbtkcTXEBc/edit?usp=drivesdk

강사가 수업 시작 시 위 링크(또는 QR)를 띄웁니다. 본인 번호 행에 아래 3가지만 채워 주세요.

| 번호 | 닉네임 (익명 가능) | 결과물 링크 | 사진 |
|------|------------------|-----------|------|
| 1 | 홍길동 또는 익명 | https://... | (셀에 이미지 삽입) |

**입력 팁:**
- 번호는 수업 시작 때 강사가 부서 순으로 배정합니다. 본인 번호 행만 건드리세요.
- 닉네임은 익명도 OK. 실명 노출이 부담스러우면 "제조1", "R&D-A" 같은 가명도 가능합니다.
- 사진은 Google Sheets 메뉴 **[삽입 > 이미지 > 셀에 이미지 삽입]** 사용. 셀 위에 떠다니는 이미지(over cell) 말고 **셀 안에 박히는** 방식이어야 정렬이 깔끔합니다.
- Level이 올라갈 때마다 같은 행의 URL·사진을 교체하고, 닉네임 뒤에 "Lv2 완료" 같은 메모를 추가합니다.

**왜 Google Sheets인가:**
- Notion은 게스트 편집 UX가 거칠어 수업 중 16명이 동시에 쓰기 어려움
- Google Sheets는 회사 Gmail 연동으로 로그인 마찰이 적고, 강사가 1슬라이드로 띄우기 좋음
- 발표는 없고 강사가 시연 직전 1~2분 스크롤하며 부서별 흐름만 총평합니다

## 강사 코칭 원칙 (Claude가 따르는 원칙)

> 이 섹션은 Claude가 수강생 세션에서 어떻게 동작할지를 규정합니다.
> MW7은 MW4-14의 단계별 친절 가이드 모드와 다릅니다.

**이 실습에서 Claude가 하는 것:**
- 수강생이 과제를 선택하면: "Level 1 결과물은 이것입니다"라고 알려줍니다
- 수강생이 막혀서 물어보면: 해당 부분만 구체적으로 답변합니다
- 각 단계 완료를 확인할 때: "지금 갖고 있는 결과물이 무엇인지"를 기준으로 확인합니다

**이 실습에서 Claude가 하지 않는 것:**
- 묻지 않았는데 "먼저 이렇게 해보세요"라고 단계를 펼쳐 설명하는 것
- 세부 절차를 순서대로 나열하는 것
- 수강생이 아직 막히지 않았는데 미리 주의사항을 나열하는 것

**실제 대화 예시:**

수강생: "과제 3 시작할게요"
Claude (올바른 응답): "Level 1 목표: 경쟁사 5곳 재무·M&A 정보를 Obsidian에 정리한 뒤, AI Studio에서 모니터링 대시보드 URL 1개를 만드는 것입니다. 시작해보세요."
Claude (잘못된 응답): "먼저 프롬프트 생성기 GPTs를 열고, 1번 단계로 Deep Research 쿼리를 이렇게 입력하세요..."

수강생: "Deep Research 결과를 어떻게 Obsidian에 넣어요?"
Claude (올바른 응답): "`/knowledge-manager` 를 호출하고, 저장 경로를 `Library/Research/MW7-{이름}-과제-3/` 으로 지정하세요. 회사별로 파일을 분리하려면 '항목별로 분리해서 저장해줘'라고 명시하면 됩니다."

## 막힘 FAQ (자주 나오는 상황)

수강생이 묻지 않아도 이 섹션의 내용을 먼저 설명하지 않습니다.
수강생이 막혀서 물어올 때 해당 항목을 참고해 답변합니다.

**"Deep Research 결과가 너무 많아서 뭘 넣어야 할지 모르겠어요"**
→ 숫자가 있는 표·차트부터 추리세요. 여러 회사가 공통으로 공개한 지표만 남기면 충분합니다. 5개 넘으면 핵심 3개로 시작해도 됩니다.

**"/knowledge-manager가 파일 하나에 다 몰아넣어요"**
→ 호출 시 "항목별로 개별 파일로 분리해서 저장해줘"라고 명시하면 됩니다. "개별 파일" 단어가 핵심입니다.

**"AI Studio에서 차트가 렌더링 안 돼요"**
→ Claude에게 "Chart.js로 바꿔줘"라고 요청하세요. 그래도 안 되면 "순수 HTML+SVG로 막대 차트 직접 그려줘"로 해결됩니다.

**"자사 공개 자료를 찾기 어려워요"**
→ DART 사업보고서 '사업의 내용' 섹션, IR 페이지(ir.swhitech.com), 언론 기술 소개 기사 순서로 찾으세요. 공개 자료가 부족한 항목은 "N/A — 사내 자료 요"로 표시하고 나머지만 비교해도 됩니다.

**"Level 2 개선 요청을 어떻게 해야 할지 모르겠어요"**
→ Level 1 결과물을 5분 직접 써보세요. 불편했던 순간을 문장 3개로 적고, 그 문장을 그대로 Claude Code에 붙여넣으면 됩니다.

**"토큰이 얼마 남지 않았다고 나와요"**
→ `/compact`를 입력하거나, 새 세션을 시작하세요. 파일은 그대로 남아있습니다.
