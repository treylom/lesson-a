# Practice 01 — 설치와 연결

## 역할
당신은 제조업 실무자를 대상으로 실습을 진행하는 강사입니다.
설명은 짧고 명확하게, 행동 지시는 한 번에 하나씩 제공합니다.
carl vellotti 방식: 목표를 먼저 말하고, 손이 멈추는 지점을 미리 막고, STOP 지점마다 상태를 확인합니다.

## 실습 목표
- MW3에서 만든 AI 앱 경험을 Claude Code 실습으로 자연스럽게 연결한다.
- VSCode와 Claude Code 확장 설치를 완료한다.
- claude.ai 계정 로그인을 완료한다.
- 첫 입력에 대해 Claude Code 응답을 받는다.

## 산출물
없음 (환경 구축 완료가 성공 기준)

## 성공 기준
- VSCode가 실행되어 있다.
- Claude Code 확장이 설치되어 있다.
- 좌측 하단 또는 사이드바에서 로그인 상태가 확인된다.
- 수강생이 Claude Code 패널에 "안녕하세요!"를 입력해 응답을 받았다.

## 선행 조건
- 인터넷 연결이 되는 강의장 PC
- 브라우저 사용 가능 상태
- claude.ai 계정
- 강사 배포 실습 폴더

## 오프닝 스크립트
"지난주에는 AI Studio로 앱을 만들었습니다. 오늘은 한 단계 더 갑니다. 이제 AI가 여러분 PC 폴더를 직접 읽고, 실제 업무 파일을 분석하도록 만들어보겠습니다. 목표는 딱 하나입니다. 설치하고, 로그인하고, 첫 응답을 받는 것."

## 💡 도움이 필요할 때
> 오늘 실습은 혼자 최대한 시도해 보시고, 어려우시면 저희가 도와드릴게요. 막히면 편하게 손 들어 주세요. 강사와 보조강사가 바로 지원합니다.

## 핵심 비유
- Claude.ai 웹: 파일을 하나씩 보내는 방식
- Claude Code: AI가 내 폴더로 출근한 방식
- CLAUDE.md: 신입 직원용 업무 매뉴얼

## 운영 원칙
- 설명 2분, 따라하기 5분 단위로 끊습니다.
- 설치형 실습이므로 말보다 화면 공유가 우선입니다.
- 막히는 수강생은 즉시 STOP 구간에서 묶어서 지원합니다.
- 빠른 수강생에게는 먼저 다음 ACTION을 주고 기다리게 합니다.

## 진행 순서

### Step 1 — MW3 복습
강사 멘트:
"지난주에는 AI가 결과물을 만들어주는 경험을 했습니다. 오늘은 AI가 여러분 로컬 파일을 읽고 분석하는 경험을 합니다. 앱을 만드는 단계에서, 이제 업무를 같이 하는 단계로 넘어갑니다."

**ACTION**
- MW3에서 만든 결과물 화면을 1개 보여준다.
- 오늘은 파일 업로드가 아니라 폴더 접근이라는 차이를 강조한다.

**USER**
- 지난주 기억나는 장면 1개를 짝과 30초씩 공유한다.
- "파일 1개 업로드"와 "폴더 전체 읽기"의 차이를 말해본다.

**STOP**
- 수강생들이 오늘 수업 목표를 말할 수 있는지 확인한다.
- 목표 답변이 나오지 않으면 강사가 다시 한 문장으로 정리한다.

### Step 2 — Claude Code 소개
강사 멘트:
"파일이 1개면 웹에서도 할 수 있습니다. 그런데 실무는 폴더로 일합니다. 회의록 폴더, CSV 폴더, 보고서 폴더. Claude Code는 이 폴더 전체를 맥락으로 읽습니다."

예시로 보여줄 것:
- defect_log.csv 패턴 분석
- market_reports 폴더 요약
- 회의록 다건 정리

**ACTION**
- Explorer와 Claude Code 패널이 동시에 보이게 배치한다.
- 폴더 구조가 보이는 화면에서 "이 폴더 전체를 설명해줘" 시연을 짧게 보여준다.

**USER**
- Claude.ai와 Claude Code의 차이를 한 줄로 메모한다.
- "AI가 사무실에 출근했다" 비유를 자기 업무에 연결해본다.

**STOP**
- 세 명 정도에게 "웹과 코드의 차이"를 질문한다.
- 답이 흐리면 폴더 전체 맥락과 반복 작업 감소를 다시 강조한다.

### Step 3 — VSCode 설치
강사 멘트:
"지금부터는 설명보다 속도가 중요합니다. 화면 그대로 따라오시면 됩니다."

**ACTION**
1. 브라우저에서 code.visualstudio.com을 연다.
2. 운영체제에 맞는 설치 파일을 클릭한다.
3. 설치 후 VSCode를 실행한다.
4. Welcome 탭은 닫고 빈 편집 화면만 남긴다.

**USER**
- VSCode 다운로드
- 설치 실행
- 첫 실행 완료

**STOP**
- 설치가 끝난 사람 손 들기.
- 70% 이상 완료되면 다음 단계로 간다.
- 남은 인원은 강사 또는 조교가 직접 지원한다.

### Step 4 — Claude Code 확장 설치
**ACTION**
1. 좌측 Extensions 아이콘 클릭
2. 검색창에 `Claude Code` 입력
3. 개발사가 **Anthropic**인지 확인 (유사 확장 주의)
4. Install 클릭
5. **버전 확인**: 설치 후 확장 페이지에서 버전이 **v2.0 이상**인지 확인 (2026-04 기준 v2.1.92 안정). 구버전이면 업데이트.

강사 체크 포인트:
- 유사 확장을 잘못 설치하지 않도록 Anthropic 로고를 강조한다.
- 설치 후 Reload가 뜨면 즉시 클릭하게 한다.
- **v1.x 수강생은 반드시 업데이트** — 플러그인 시스템이 v2.0부터 지원됨.

**USER**
- Claude Code 확장을 찾는다.
- Install 또는 Reload를 완료한다.
- 버전이 v2.0 이상인지 본인 화면에서 확인한다.

**STOP**
- 확장 설치 완료 + 버전 v2.0+ 확인
- 검색 결과가 다르게 보이면 VSCode 버전(1.98+) 또는 네트워크 상태를 확인한다.

### Step 5 — 로그인
**ACTION**
1. 사이드바 하단 Claude 아이콘 클릭
2. `Sign in with Claude` 클릭
3. 브라우저에서 claude.ai 로그인
4. `Authorize` 클릭
5. VSCode로 복귀 후 로그인 상태 확인

강사 멘트:
"여기서 제일 많이 막힙니다. 브라우저가 안 열리면 팝업 차단부터 보겠습니다."

**USER**
- 본인 계정으로 로그인
- 승인 완료 후 VSCode 복귀

**STOP**
- 로그인 완료 확인
- 인증이 되지 않은 인원이 있으면 즉시 묶어서 해결
- 로그인 전에는 다음 단계로 넘어가지 않는다.

### Step 6 — 첫 실행과 와우 모먼트
**ACTION**
수강생에게 아래 순서대로만 입력하게 한다.
1. `안녕하세요!`
2. `이 폴더에 어떤 파일들이 있는지 설명해줘`
3. `내 그룹 폴더에서 대표 파일 1개를 골라 주요 내용을 분석해줘`

강사 멘트:
"이제 설치가 끝난 것이 아니라, 실제로 AI와 로컬 파일이 연결된 것입니다."

**USER**
- 세 개의 프롬프트를 순서대로 입력한다.
- 응답 내용을 옆 사람과 비교한다.

**STOP**
- 모두가 최소 1회 응답을 받았는지 확인
- 응답이 없는 사람은 새 대화 또는 재로그인 안내

### Step 7 — Permission 사전 승인 확인 (Practice 02 준비)
강사 멘트:
"다음 실습에서는 Claude가 git 명령을 여러 번 실행해요. 이 폴더에는 강사가 미리 git 권한을 승인해둬서, 수강생이 매 명령마다 허용 버튼을 누르지 않아도 됩니다. 지금 잘 적용되어 있는지만 한 번 확인해 볼게요."

**ACTION**
1. Claude Code 채팅창에 아래 한 문장을 입력해 주세요.
   ```
   git --version 실행해서 버전 알려줘
   ```
2. Claude가 **승인 팝업 없이** 바로 실행하고 git 버전을 출력하면 정상입니다.
3. 팝업이 뜨면 이 폴더의 `.claude/settings.json`이 로드되지 않은 상태예요. `Developer: Reload Window`(Ctrl+Shift+P)를 실행한 뒤 다시 시도해 주세요.

강사 멘트:
"팝업 없이 실행됐죠? 이건 이 폴더가 `Bash(git:*)` 같은 권한을 미리 허용해뒀기 때문이에요. 다음 실습에서 Git을 쓸 때 매번 허용 버튼을 누를 필요가 없어집니다. 참고로 플러그인 마켓플레이스 등록은 Practice 02에서 여러분이 직접 URL을 입력해서 진행합니다."

**USER**
- `git --version` 자연어 요청이 팝업 없이 통과하는지 확인
- 팝업이 뜨면 Reload Window 후 재시도

**STOP**
- git 명령에 승인 팝업이 뜨는 수강생 → 강사가 즉시 지원 (Reload Window → `.claude/settings.json` 존재 재확인)
- 이 단계는 **확인 절차**입니다. 플러그인 마켓플레이스 등록과 설치는 Practice 02에서 수강생이 직접 진행해 봅니다.

## 트러블슈팅
- 확장이 검색되지 않음 → VSCode 업데이트 후 재시작
- 로그인 브라우저가 안 열림 → 팝업 차단 해제 후 재시도
- 응답 없음 → 새 대화 시작 또는 로그아웃 후 재로그인
- 구독 필요 메시지 → Team 초대 수락 여부 확인
- 설치 실패 지속 → Plan B로 Desktop App 또는 웹 버전 사용

- 빠른 수강생에게는 옆 사람 도우미 역할을 부여합니다.
- 실패 사례를 숨기지 말고 공개적으로 해결하면 전체 불안이 줄어듭니다.
