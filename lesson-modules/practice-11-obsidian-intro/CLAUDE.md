### Step 1 — "왜 자료를 정리해야 하는가" + 핵심 단어 (5-15분)

**강사 멘트 (5분)**:

> "질문 하나 드릴게요.
> 여러분 컴퓨터에 파일이 몇 개 있는지 아세요?
>
> 보통 직장인 PC에 문서만 수천 개가 있어요.
> 엑셀, 한글, PDF, 사진, 메모...
> 이걸 AI에게 '읽어봐'라고 하면 어떻게 될까요?
>
> AI는 — **읽을 수는 있는데, 어디서부터 읽어야 하는지 모릅니다.**
>
> 집을 생각해보세요.
> 서류가 책상 위에 마구 쌓여있으면 → '금형 도면 어디 있지?' 찾는 데 10분
> 서류가 서랍장에 '도면 / 보고서 / 메모' 정리되어 있으면 → 3초
>
> AI도 똑같아요.
> 정리 안 된 자료 → 느리고 대답 품질 낮음
> 정리된 자료 → 빠르고 대답 정확함
>
> 오늘부터 우리가 할 건 **AI에게 '서랍장'을 만들어주는 것**이에요.

>
> 좋은 소식은 — 정리 자체도 AI가 도와줘요.
> 3교시에 가면 AI가 자동으로 정리하는 장면을 직접 볼 겁니다."

**핵심 단어 4개 칠판에 띄우기 (화면 공유)**:

> "핵심 단어 4개만 기억해주세요.
> ① **vault** = 노트 모음 폴더. 그냥 폴더예요.
> ② **그래프** = 노트들이 어떻게 연결되어 있는지 보여주는 그림
> ③ **wikilink** = 노트끼리 손을 잡는 것. `[[다른 노트 제목]]` 이렇게 쓰면 연결돼요. 대괄호 2개는 '이 단어를 다른 노트와 연결해달라'는 표시예요. AI가 이걸 보면 자동으로 관련 노트를 찾습니다.

> ④ **백링크** = '이 노트를 인용한 다른 노트들' 자동 표시.

>

> 1. VSCode 왼쪽 탐색기(Explorer) 상단에 폴더 이름이 보입니다

### Step 3 — AI가 .md를 사랑하는 이유 (28-35분)

**강사 멘트 (5분)**:

> "왜 하필 .md 파일일까요?
>
> .md는 Markdown 파일이에요. 메모장(.txt)이랑 거의 같은데, 한 가지 다른 점이 있어요.
> **구조가 있다**는 거예요.
>
> 이렇게 생겼어요 (화면 공유):
> ```
> # 제목
> ## 소제목
> - 항목 1
> - 항목 2
> **강조**
> ```
>
> 왜 이게 중요하냐면:
> 1. **AI가 읽기 편해요** — '#'이 제목인 걸 알고, '-'가 리스트인 걸 알아요.
>    엑셀이나 한글 파일은 AI가 열기도 힘들어요.
> 2. **가벼워요** — .md 파일 1개가 보통 1-10KB. 엑셀 1개가 보통 500KB-5MB.
>    AI한테 1000개 .md 파일 주는 것 vs 1000개 엑셀 주는 것, 뭐가 빠를까요?
> 3. **어디서든 열려요** — 메모장, VSCode, 옵시디언, 심지어 GitHub에서도.
>    '이 파일 열려면 뭘 설치해야 해요' 같은 문제가 없어요.
>
> 정리하면: **AI에게 .md는 모국어**예요. 다른 형식은 외국어고요.
> 여러분이 AI에게 일 시킬 때, .md로 자료를 주면 AI가 훨씬 잘해요."

**실습 (2분)**: 수강생이 vault에서 아무 .md 파일 하나를 열어서 내용 확인.

> "아무 파일이나 클릭해보세요. 열어서 보이는 게 바로 Markdown이에요. # 있고, - 있고, [[]] 있는 게 보이시죠?"

### Step 4 — 노션 vs 옵시디언 라이브 비교 (35-42분)

#### 4a. "왜 옵시디언인가" — 속도·비용·보안 3가지 이유 (3분)

**강사 멘트 (핵심 포인트)**:
> "여러분 질문 하나. 왜 하필 옵시디언일까요?
> 회사에 이미 팀즈, SharePoint, 노션, 구글 드라이브 있잖아요.
>
> 이유는 딱 3가지예요: **속도, 비용, 보안**.
>
> **① 속도** — 옵시디언은 내 PC 폴더예요.
> 팀즈나 노션은 인터넷 서버에 파일이 있어서, AI가 읽으려면 서버까지 갔다 와야 해요.
> 옵시디언은 그냥 내 컴퓨터 폴더니까 **바로 읽어요**. 50배 이상 빨라요.
>
> **② 비용** — 팀즈나 노션에서 AI가 파일 읽으려면 **API 설정이 복잡**해요.
> 회사 IT팀한테 부탁해야 하고, 설정이 잘못되면 안 돌아가요.
> 옵시디언은 설정 필요 없이 Claude Code가 **바로 접근 가능**. 시간이 곧 비용이잖아요.
>
> **③ 보안** — 이게 의외로 중요해요.
> 노션에 저장하면? 파일이 **노션 서버(미국)**에 있어요.
> 팀즈에 저장하면? 파일이 **마이크로소프트 서버**에 있어요.
> 옵시디언에 저장하면? 파일이 **내 PC에만** 있어요. 인터넷에 안 올라가요.
>
> 회사 보안 정책 생각해보세요. 금형 도면이나 불량 데이터를 외부 서버에 올리면 안 되죠?
> 옵시디언은 **내 컴퓨터에서 안 나가요**. 가장 안전한 방법이에요.
>
> 정리: **빠르고, 설정 안 복잡하고, 보안까지 안전한** 게 옵시디언이에요."

**비교 표 (화면에 띄우기)**:

| 기준 | 팀즈/SharePoint | 노션 | 옵시디언 |
|------|----------------|------|---------|
| 파일 위치 | 외부 서버 (미국) | 외부 서버 (미국) | **내 PC에만** |
| AI가 파일 읽는 속도 | 느림 (서버 왕복 3-5초) | 느림 (서버 왕복 3-5초) | **즉시** (0.1초) |
| API 연결 | 복잡 (IT팀 도움 필요) | 복잡 (별도 설정 필요) | **없음** (바로 됨) |
| 오프라인 사용 | 불가능 | 불가능 | **가능** |
| 보안 | ⚠️ 외부 서버에 저장 | ⚠️ 외부 서버에 저장 | ✅ **PC에서 안 나감** |

> "제조업에서 AI 쓸 때 가장 많이 받는 질문이 '보안 괜찮아요?'예요.
> 옵시디언이면 — **네, 내 컴퓨터에서 안 나갑니다**. 이 한마디면 돼요."

#### 4b. 라이브 시연 (5분)

**강사 시연 (화면 공유)**:

```
시연 1: 옵시디언에서 student-profile.md 열기 → 즉시 열림
시연 2: 같은 내용을 노션 페이지에서 열기 → 로딩 시간 체감
시연 3: Claude Code에서 "student-profile.md 읽어줘" → 같은 폴더이므로 즉시 읽음
```

> "보셨죠? Claude Code가 같은 폴더에 있는 파일을 바로 읽어요.
> 노션이었으면 '노션 API 설정하세요, MCP 연결하세요' 이런 과정이 필요해요.
> 옵시디언은 그냥 폴더니까 → Claude Code가 바로 읽어요.
>
> 이게 '**AI가 정보를 읽는 방법**'의 핵심이에요.
> 가장 쉬운 경로 = 폴더 + .md + 옵시디언."
