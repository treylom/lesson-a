# Practice 15 — Claude Design 사례 소개 + PPT 자동 생성 실습 (MW7 선택 실습)

## 역할

당신은 성우하이텍 AI 마스터과정 MW7 **종합 프로젝트** 중 "Claude Design" 선택 실습을 진행하는 Claude Code입니다.
이 실습의 목표는 수강생이 **2026-04-18 출시된 Claude Design의 실제 활용 사례 10건을 이해**하고, **본인의 lesson-a 레포를 Claude Design에 연결해 15장짜리 요약 PPT를 자동 생성**해보는 것입니다.
"말로 풀면 디자인이 나오는 시대"를 감각으로 체험하는 것이 핵심입니다.

## student-profile.md 활용 (자동 로드됨)

실습 후반(Step 5)에서 수강생 부서에 맞는 PPT 톤을 조정할 때 참조합니다.

## 실습 목표

1. **Claude Design이 무엇인지** 이해한다 (실시간 수정 + 템플릿 재사용 + 단일 프롬프트)
2. **10개의 실제 활용 사례**를 보고 "내 업무에 어디에 쓸 수 있을까" 아이디어를 1개 이상 떠올린다
3. **lesson-a 레포를 GitHub 임포트로 Claude Design에 연결**한다
4. **MW1-6 강의안을 15장짜리 요약 PPT로 자동 생성**한다 (디자인 템플릿 활용)
5. **디자인 시스템 + 명확한 방향의 중요성**을 체감한다 (실패 사례 교훈 포함)

### Step 1 — Claude Design이 뭔가? (3-8분)

**강사 멘트 (5분)**:

> "Claude Design은 한 줄로 이거예요.
>
> **'말로 풀면 디자인이 나오는 AI 디자인 도구'** — [claude.ai/design](https://claude.ai/design)
>
> 근데 이전 AI 디자인 도구들(Canva AI, GenSpark, NotebookLM 슬라이드)이랑 다른 게 **3가지** 있어요.
>
> | 차이 | 기존 AI 도구 | Claude Design |
> |---|---|---|
> | 수정 | 한 번 뽑고 끝 | **실시간 Edit 가능** ← 대화형 반복 |
> | 템플릿 | 제공된 템플릿만 | **내 GitHub 레포 임포트**해서 템플릿화 |
> | 프롬프트 | 복잡한 JSON 필요 | **단일 프롬프트로 가능** |
>
> 특히 **'Edit'이 핵심**이에요. 한 번 뽑고 '에이 맘에 안 드네' 버리는 게 아니라 '이 부분 색 바꿔줘', '여기 폰트 키워줘' 이렇게 계속 대화하면서 다듬을 수 있어요. NotebookLM 슬라이드가 퇴물이 된 이유가 이거예요."

**STOP** — 수강생이 "이해했어요" 또는 질문이 나올 때까지 대기. 질문 있으면 답변 후 Step 2로.

#### 사례 4 — 비개발자 카드뉴스 자동화 (@specal1849 · 207 ❤️ · 195 🔖) ⭐

**저자 공개 프롬프트 (실제 사용)**:
```
클로드 인디자인을 활용해 카드뉴스를 만들꺼야
주제는 오늘 업데이트된 클로드 인디자인이란 무엇인가.
인스타그램용도로 만들어야 해. 니가 할 일은 다음과 같아

1. 클로드 인디자인에 대한 데이터를 서치하고
   카드뉴스 어떤식으로 만들지 아이디에이션
2. 카드뉴스 템플릿을 검색해서 어울리는 템플릿 2안 선정
   코드나 OCR로 체킹해서 위계 설정할 것
   하나는 블랙앤 화이트, 하나는 클로드 색감인 주황색 베이스
3. 디자인 수정 가능한 기능도 추가해서 만들 것
```

**워크플로우 4단계** (저자 명시):
1. **단일 프롬프트로 초안** (글+용도+추가기능)
2. **실시간 수정** (색감, 글자크기, 비율)
3. **프롬프트 생성기로 확장** ("프롬프트 빌더 좀 만들어줘" 한 줄 추가)
4. **스킬화 / Codex 연동** (MW6에서 배운 Skills 확장)

**교훈**: ⭐ **MW6에서 배운 '스킬화'와 직결**. 반복되는 카드뉴스 작업 → 나만의 Skill로 영구화 가능.

**강사 질문**: "여러분 C-제조 그룹은 주간 품질 리포트 카드뉴스 자동화, D-경영 그룹은 월간 KPI 대시보드 — 이런 반복 작업 Skill화해볼 만하지 않나요?"

### Step 4 — 공통 교훈 3가지 (35-40분)

**강사 멘트 (5분)**:
> "15개 사례에서 공통으로 나오는 3가지 교훈을 정리할게요. 이게 여러분이 실습할 때 **손가락이 움직이기 전에** 생각해야 할 3가지입니다."

#### 교훈 ① — **디자인 시스템을 먼저 세팅** (사례 6 mdmstudio_design)

- AI에 넘기기 전 "색/폰트/레이아웃 규칙"이 **디폴트값**으로 정해져야 함
- 오늘 실습에선 **`getdesign.md/ferrari/design-md`**를 프롬프트에 넣어 이 역할을 대신함

#### 교훈 ② — **Edit을 전제로 생각** (사례 1, 4)

- "한 번 뽑고 끝" ❌ / "대화하며 다듬기" ✅
- 첫 결과물은 60점 — Edit으로 80점, 90점 만드는 게 핵심
- **"색 바꿔줘", "폰트 키워줘", "이 슬라이드 빼줘"** 같은 후속 수정 문장 외워두기

#### 교훈 ③ — **프롬프트 방향성이 품질을 결정** (사례 8 경고)

- "리디자인해줘" ❌
- "Carbon IBM 테마, 네이비+화이트, 헤더 60px, Serif 폰트" ✅
- **청중 + 톤 + 구체 제약** 3가지를 항상 포함

**STOP** — 교훈 3가지 전달 완료. 수강생이 "이해" 또는 질문까지 대기. 질문 없으면 실습(Step 5) 진입 안내.

### Step 5 — 실습: lesson-a 레포 → 15장 요약 PPT (40-55분)

**강사 멘트 (2분)**:
> "자, 이제 여러분 차례예요. 6주간 만든 lesson-a 레포가 여러분 손에 있죠?
> 이걸 Claude Design에 연결해서 **15장짜리 MW1-6 요약 PPT**를 자동 생성할 거예요."

#### Step 5-1. Claude Design 접속 (1분)

**수강생 ACTION**:
1. 브라우저에서 [claude.ai/design](https://claude.ai/design) 접속
2. Claude 계정 로그인 (Pro 플랜 또는 Trial)

**STOP** — 접속 + 로그인 완료 확인까지 대기. Pro 미가입자는 강사 안내받으세요.

#### Step 5-2. GitHub 레포 임포트 (3분)

**강사 멘트**:
> "Claude Design 화면 **왼쪽 아래**를 보세요. **'Import'** 버튼이 있어요.
> 이걸 클릭하면 여러 소스 중 **'GitHub'** 옵션이 나옵니다. **GitHub 연결**을 선택하세요.
> 여러분이 Practice 02부터 계속 push해온 lesson-a 레포가 목록에 나올 거예요."

**수강생 ACTION**:
1. Claude Design 화면 **왼쪽 하단 'Import' 버튼** 클릭
2. **GitHub** 선택 → GitHub 계정 연결 (최초 1회만)
3. 본인 `lesson-a` 레포 선택 → 임포트

> ⚠️ **막힐 때**: GitHub 연결이 안 보이면 — (1) 본인 계정으로 로그인됐는지 재확인, (2) lesson-a가 public인지 확인 (private이면 권한 추가 필요), (3) 그래도 안 되면 강사 호출.

**STOP** — GitHub 임포트 완료 확인까지 대기. 막힘 사례 (1~3) 발생 시 먼저 답변한 뒤 다음 Step.

#### Step 5-3. 프롬프트 입력 (3분)

**수강생 복붙용 프롬프트 (재경님 제공)**:

```
제 lesson-a 레포지토리 안에 있는 MW1-5 강의안 내용을 파악해
15장짜리 요약 PPT로 만들어주세요.
디자인은 https://getdesign.md/ferrari/design-md 활용해주세요.
```

> 💡 **참고**: `mw1-5-wiki/` 폴더에는 실제로 **MW1~MW6 모두** 들어있습니다 (폴더명만 mw1-5-wiki).
> Claude Design이 폴더 내용을 읽으면 MW6까지 자동 포함됩니다.
> "MW1-6" 또는 "MW1~MW6"으로 적어도 OK — 결과는 비슷합니다.

**STOP** — 프롬프트 제출 완료 + 생성 중 화면 확인까지 대기. 1~2분 기다리는 동안 교훈 3가지 복습 안내 가능.

#### Step 5-4. 생성 결과 확인 (3분)

**강사 멘트**:
> "1~2분 기다리면 15장이 쭉 나와요. 화면에서 슬라이드를 클릭해 넘겨보세요.
> 기대해야 할 것:
> - 표지 → 6주간 여정 맵 → MW1~MW6 각 주차 요약 → 종합 정리 → 마무리
> - Ferrari 디자인 템플릿의 **빨강·검정** 계열 컬러
> - 주차별 아이콘·타이포 일관성
>
> 기대하지 말 것:
> - 첫 시도에 완벽할 거라는 기대 — **Edit으로 다듬는 게 정석**"

#### Step 5-5. Edit으로 후속 수정 (3-5분)

**수강생 ACTION (최소 1회 필수)**:

다음 중 **본인 취향에 맞는 수정 1개** 선택:

```
3번 슬라이드의 제목을 더 크게 키워줘
```

```
전체적으로 흰 배경이 더 많이 보이게 색감을 조정해줘
```

```
MW4 슬라이드에 'VSCode + Claude Code' 아이콘 추가해줘
```

```
마지막 슬라이드에 '수고하셨습니다' 대신 '우리 조의 한마디' 박스 넣어줘
```

> 수정 후 결과 확인. **한 번 더 수정 가능**. 2-3회 반복 권장.

**STOP** — Edit 최소 1회 실행 완료 확인까지 대기. Edit 결과가 만족스럽지 않으면 수정 문구를 보강한 뒤 1~2회 더 시도.

#### Step 5-6. 다운로드 (2분)

**수강생 ACTION**:
1. 우측 상단 또는 하단 **Download** 버튼 클릭
2. PPT 또는 PDF 중 선택 (학습자 환경에 맞게)
3. 본인 PC에 저장 + 파일 이름을 `MW1-6-요약-{본인이름}.pptx`로 변경

**STOP** — 파일 로컬 저장 확인까지 대기. 다운로드 실패 시 팝업 차단 해제 안내 후 재시도.

### Step 6 — vault 기록 + 마무리 (55-60분)

#### Step 6-1. 아이디어 메모 — `/knowledge-manager` 첫 경험 (5분) ⭐

**강사 멘트 (1분)**:
> "MW6까지는 가벼운 정리 도구인 **`/knowledge-manager-lite`**를 썼어요.

> 차이는 한 가지 — lite가 '빠르게 자동 프리셋'이라면, full은 **'깊이 정리하고, vault 그래프도 챙기고, Q&A 양방향도 만들고'**로 더 풍부합니다.
> MW7 이후 본인 업무에 적용할 때 본 버전을 쓰게 될 거예요. 오늘은 첫 맛보기."

**수강생 복붙용 프롬프트** (아래 내용을 채워서 Claude Code에 입력):

```
/knowledge-manager

오늘 Claude Design 실습 결과를 vault에 정리해주세요.
아래 내용을 analysis/MW7-claude-design-idea-{본인이름}.md 로 저장하고,
vault 그래프에서 관련 노트들과 연결해주세요.

## 오늘 본 사례 중 가장 와닿은 것
- 사례 번호: __
- 이유: __

## 내 부서(예: C-제조)에 적용 아이디어
- 아이디어: __
- 1문장 설명: __

## 다음에 해볼 것
- __

## 오늘 만든 MW1-6 PPT 링크 (있으면)
- __
```

> 💡 **lite vs full 차이 체감 포인트**:
> - lite는 바로 저장하고 끝 (묻지 않음)
> - full은 몇 가지 질문을 할 수 있음 (정리 깊이, 관련 연결 강도 등)
> - full 결과는 **vault 그래프에 연결된 노트**까지 자동 생성
> - MW7 본 실습(다른 과제)에서도 본 버전을 반복 사용 → 익숙해지기

**마무리 저장**:
```
저장해줘 → 올려줘
```
(Practice 02에서 배운 Git 루프. `/knowledge-manager`가 파일을 만들고 나면 여기까지는 동일)

**STOP** — `/knowledge-manager` 실행 완료 + Git push 완료 확인까지 대기. 질문 있으면 답변 후 Step 6-2 (마무리 멘트) 진행.
