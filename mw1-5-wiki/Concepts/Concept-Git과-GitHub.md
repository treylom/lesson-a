# Git과 GitHub — 파일을 안전하게 저장하고 공유하는 방법

## 한 줄 요약

Git은 **파일 변경 이력을 기록하는 시스템**, GitHub은 **그 이력을 인터넷에 저장하는 곳**입니다.

## 쉬운 비유

- **Git** = 문서의 "변경 이력 추적" (한글의 "변경 내용 추적"과 비슷)
- **GitHub** = 변경 이력이 포함된 문서를 저장하는 "팀 공유 드라이브"

## 핵심 용어 3개

| 용어 | 의미 | 일상 비유 |
|------|------|----------|
| commit | 변경 내용을 기록 | "저장 + 메모 남기기" |
| push | 기록을 GitHub에 올림 | "팀 드라이브에 업로드" |
| pull | GitHub에서 최신 내용 받기 | "팀 드라이브에서 다운로드" |

## "저장해줘 → 올려줘" 루프

수업에서 매번 하는 이 루프가 바로 Git 사용입니다:

```
"저장해줘" = 파일을 만들고 commit
"올려줘"   = GitHub에 push
```
