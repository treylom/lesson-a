### 핵심 발견

- **[[Concept-CLAUDE-md]] = AI의 업무 규칙서**: "항상 표로 정리해줘" 같은 규칙을 파일에 저장
- **[[Concept-프롬프트]] 4법칙**: 역할 + 맥락 + 형식 + 제약
- **[[Concept-워크플로우]] = 반복 가능한 작업 순서**: 매주 같은 분석을 자동화
- **서브에이전트 첫 체험**: deep-research로 AI가 다른 AI를 불러서 검색

### CLAUDE.md 규칙 예시

```markdown
## Always (항상 하기)
- 결과는 표 형식으로 정리
- 한국어로 답변

## Ask (물어보기)
- 파일을 삭제할 때

## Never (절대 하지 않기)  
- 개인정보 포함된 파일 외부 전송
```

### 프롬프트 4법칙

| 법칙 | 예시 |
|------|------|
| 역할 | "너는 제조팀 품질 분석가야" |
| 맥락 | "이 CSV는 지난달 불량 데이터야" |
| 형식 | "표로 정리하고, 상위 3개만" |
| 제약 | "전문 용어는 쉽게 풀어서" |

### 바이브 코딩 (Vibe Coding)

"바이브 코딩"은 코드를 직접 작성하지 않고 AI에게 자연어로 설명만 해서 원하는 결과물을 만드는 방식입니다.
예를 들어 "이 CSV를 분석해서 불량률 추이 그래프를 그려줘"라고 말하면, AI가 코드를 작성하고 실행까지 해줍니다.

MW5에서 배운 [[Concept-워크플로우]]와 [[Concept-프롬프트]] 4법칙이 바로 바이브 코딩의 핵심입니다.
코딩을 모르는 제조업 실무자도 AI에게 정확히 설명할 수만 있으면 개발자와 같은 결과물을 만들 수 있습니다.

| 기존 방식 | 바이브 코딩 |
|----------|-----------|
| 엑셀 함수 직접 작성 | "매출 상위 5개 제품을 찾아서 표로 정리해줘" |
| Python 코드 작성 | "이 데이터에서 불량 패턴을 찾아줘" |
| 보고서 양식 수동 작성 | "팀장님 보고용으로 바꿔줘" |
